For a while, I've wanted to set up a coding school. People often ask me why, and the answer is simple: I feel like I should help people, and I feel like I can.

Less clear are the specifics, for example: what course material, what lesson format, how often should it run, etc.

On 2017-06-13 I released a survey to my friends and family on Facebook, and to my coworkers at The App Business where I'm currently contracting. If you'd like to fill it out, you can see it here: https://goo.gl/forms/6vM9SNYQJqq4RdSL2

Mentors and Students

Are you interested in being a student or mentor?

As an incredibly rough guess, I would estimate that I would need about 1 instructor per 5 students, depending on course format.

This is a great ratio! If I could get this proportion of people to attend this would be perfect.

Course Material

What would you like to learn about? (if you want to mentor, answer which you feel like you would be able to mentor)

The options were:

  • How to make a mobile app from scratch
  • How to make a website from scratch
  • How to make physical things e.g. electronics, soldering
  • How the Internet works
  • How computers and computer programs work

This was pretty close, with apps getting 22 and websites getting 20. I'm much more comfortable with websites than mobile apps, so I'm going to be teaching how to make a website for the first set of sessions. There's definite interest for apps, so it's possible that the second set of sessions will focus on this.

I will also cover how the Internet works, and to some extent how computers and computer programs work. Unfortunately there weren't enough instructors to teach making electronics, so I won't be covering this for now.

There were also a few write-ins, notably for Python, WordPress and Twitter Bots.

Pre-existing ideas for what to build

Is there anything you already know you'd like to build? If so, what is it?

This was a pure write-in question. Most of these were either website or app based, some with a specific business use in mind. One was to build a portfolio website for themself as a freelancer, which is very doable as a project!

Course format

What course format would you prefer?

The options were:

  • One set course, each session building on the knowledge of previous session
  • Many smaller courses, each one spanning a few sessions
  • Standalone lessons or workshops
  • No fixed agenda, bring your own projects and get help from the instructors.

There is no clear winner here! The one set course is slightly more popular, and it's what I feel more comfortable teaching, so it's the one I'll go with initially. I'll try to break it down into smaller chunks that are useful on their own.

Session format

What session format would you prefer?

The options were:

  • Lectures/talks for entire session
  • Short lecture at beginning to introduce concepts, followed by working on exercises with help from instructors
  • Social, no fixed structure other than instructors are there to help

This one has a clear winner, labs-style workshops, which I also agree with.

Focus on people who are under-represented in tech

Some other organisations (e.g. Code First: Girls, Code Bar, Trans*Code) focus on people who are under-represented in tech, would something like this be positive/negative for you?

I have a lot of admiration for these other organisations, and wanting to help people is a big part of why I'm doing this. I wanted to see if other people felt the same way.

I broke it down into either making the event exclusively for people who self-define as underrepresented in tech, having some support, and doing nothing. I asked people if they felt positive/neutral/negative about those options.

Proportion of people who identify as under-represented in tech

Do you identify as under-represented in tech?

I didn't focus on a particular way people could be underrepresented because while some are very visible (e.g. gender, race), some are less so (e.g. wealth, sexual orientation), and some can be not at all (e.g. mental health, some physical disabilities). It would be impossible for me to list them all and I don't want to unintentionally erase anyone, so I let people self-define.

I'm going to discuss all of these as there were some good points raised.

Some people were shy or worried about attending something like this. I'm hoping that being this open and transparent will help!

Findings

I'm grateful for all the feedback I received. It's been really useful to confirm some theories I had about how I wanted to run this.

To summarise what I'd like to do: one course of 6-8 sessions with individual sessions that would be useful as standalone or in small subsets. Free event but with ticketing (the capacity depends on venue) and a percentage (possibly two thirds) of the tickets being only for people who identify as under-represented in tech. Put course material online and figure out how to develop from a Chromebook.

I'm picturing a set of lessons like this (although I'm open to ideas!):

  • intro + concepts + starter HTML
  • more HTML + starter CSS
  • Twitter Bootstrap + responsiveness
  • basic source control with Git + GitHub
  • intro to JavaScript and jQuery
  • more JavaScript and jQuery

If there's something else I should be thinking about, or if you have comments or opinions, please email robbie@hackathing.io
